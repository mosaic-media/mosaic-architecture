# Language

Mosaic Design Language specifications describe product philosophy, mental models, interaction behaviour, composition, and governance.

## Design Language in Brief

Read the Design Language in order: each specification adds a layer of intent used by the next.

| Specification | In one sentence |
|---------------|-----------------|
| [MDL-001 — Mosaic Design Language Vision](mdl-001-vision/index.md) | Defines why Mosaic exists and the product beliefs that govern its direction. |
| [MDL-002 — Principles](mdl-002-principles/index.md) | Turns the vision into a framework for making consistent design decisions. |
| [MDL-003 — Mental Model](mdl-003-mental-model/index.md) | Defines the conceptual world that Mosaic and its contributors should understand. |
| [MDL-004 — Interaction Model](mdl-004-interaction-model/index.md) | Defines how that world changes over time while preserving continuity and context. |
| [MDL-005 — Composition Model](mdl-005-composition-model/index.md) | Defines how information and relationships become coherent, prioritised experiences. |

The linked MDL specifications own these concepts; this page is a reading guide rather than a substitute for them.
